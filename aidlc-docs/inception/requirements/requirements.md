# 테이블오더 서비스 - 요구사항 문서

## Intent Analysis

- **User Request**: 테이블오더 서비스 구축 (고객 주문 + 관리자 대시보드)
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: System-wide (프론트엔드 + 백엔드 + DB + 인프라)
- **Complexity Estimate**: Complex (다중 사용자 유형, 실시간 통신, 세션 관리, 멀티 테넌트)

---

## 기술 스택 결정사항

| 영역 | 기술 | 비고 |
|---|---|---|
| 백엔드 | Node.js (Express/Fastify) | REST API + SSE |
| 프론트엔드 | Vue.js | 단일 앱, 라우팅으로 고객/관리자 분리 |
| 데이터베이스 | PostgreSQL | 관계형 DB |
| 배포 환경 | Docker Compose | 로컬 개발 환경 (프로덕션 배포는 추후 결정) |
| 이미지 저장 | 서버 로컬 파일 시스템 | 업로드 기능 포함 |

## Functional Requirements

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지 (JWT 기반)

### FR-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 리다이렉트
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 이용 완료 처리된 주문 제외

### FR-06: 매장 관리자 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션 유지
- bcrypt 비밀번호 해싱
- 로그인 시도 제한
- 브라우저 새로고침 시 세션 유지

### FR-07: 실시간 주문 모니터링
- SSE(Server-Sent Events) 기반 실시간 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

### FR-08: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호, 16시간 세션)
- 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- 테이블 세션 종료 (이용 완료): 주문 내역 → 과거 이력 이동, 현재 주문/총액 리셋
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

### FR-09: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지)
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

### FR-10: 초기 데이터 및 관리자 계정
- 시드 데이터: 샘플 매장, 메뉴, 테이블 데이터 포함
- 기본 관리자 계정 자동 생성 (시드 데이터)

## Non-Functional Requirements

### NFR-01: 확장성
- 대규모 SaaS 지원 (100개+ 매장)
- 매장당 30~100개 테이블 지원
- 멀티 테넌트 아키텍처 고려

### NFR-02: 성능
- SSE 기반 실시간 주문 알림 2초 이내
- 메뉴 조회 응답 시간 최소화

### NFR-03: 보안 (Security Extension 적용)
- SECURITY 규칙 전체 적용 (blocking constraint)
- 암호화 at rest / in transit
- bcrypt 비밀번호 해싱
- JWT 토큰 기반 인증
- 로그인 시도 제한

### NFR-04: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 레이아웃
- 명확한 시각적 계층 구조

### NFR-05: 데이터 무결성
- 장바구니 로컬 저장 (새로고침 유지)
- 세션 기반 주문 격리
- 주문 이력 영구 보존 (OrderHistory)

### NFR-06: 개발 환경
- Docker Compose 기반 로컬 개발 환경
- PostgreSQL 컨테이너
- Node.js 백엔드 컨테이너
- Vue.js 프론트엔드 (개발 서버)

---

## 제외 범위 (Constraints)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석 대시보드, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도/번역 API)

## 테이블 세션 라이프사이클

```
새 고객 착석 → 첫 주문 시 세션 시작 → 추가 주문 가능 → 관리자가 이용 완료 처리 → 세션 종료 (주문 이력 이동) → 테이블 리셋 → 다음 고객 대기
```
