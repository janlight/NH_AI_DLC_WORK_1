# Business Rules - Unit 2: menu-management

## 1. 카테고리 규칙

### BR-CAT-01: 카테고리명 유일성
- 동일 매장(storeId) 내에서 카테고리명은 중복 불가
- 대소문자 구분 없이 비교 (예: "음료" == "음료")
- 앞뒤 공백 제거(trim) 후 비교

### BR-CAT-02: 카테고리 삭제 제약
- 해당 카테고리에 활성 메뉴(isActive=true)가 존재하면 삭제 불가
- 에러 메시지: "해당 카테고리에 활성 메뉴가 있어 삭제할 수 없습니다."
- 모든 메뉴를 다른 카테고리로 이동하거나 비활성화 후 삭제 가능

### BR-CAT-03: 카테고리 순서 관리
- sortOrder는 0 이상의 정수
- 새 카테고리 생성 시 sortOrder 미지정이면 현재 최대값 + 1 자동 할당
- 순서 변경 시 배열로 전체 순서를 한 번에 업데이트

### BR-CAT-04: 카테고리 소속 검증
- 카테고리 CRUD 시 storeId가 요청자의 매장과 일치하는지 검증
- 다른 매장의 카테고리 접근 시 404 반환 (정보 노출 방지)

---

## 2. 메뉴 규칙

### BR-MENU-01: 필수 필드 검증
- 메뉴 등록 시 필수: name, price, categoryId
- name: 1~100자, 공백만으로 구성 불가
- price: 정수, 100 ≤ price ≤ 500,000
- categoryId: 해당 매장에 존재하는 활성 카테고리여야 함

### BR-MENU-02: 메뉴명 유일성
- 동일 매장, 동일 카테고리 내에서 활성 메뉴명 중복 불가
- 비활성(soft deleted) 메뉴는 중복 검사에서 제외
- 앞뒤 공백 제거(trim) 후 비교

### BR-MENU-03: 가격 검증
- 타입: 정수 (소수점 불가)
- 범위: 100원 이상, 500,000원 이하
- 범위 초과 시 에러: "가격은 100원 이상 500,000원 이하여야 합니다."

### BR-MENU-04: Soft Delete
- 메뉴 삭제 시 물리적 삭제가 아닌 isActive = false로 변경
- 고객 메뉴 조회 시 isActive = true인 메뉴만 반환
- 관리자 메뉴 관리 시 isActive 상태 표시 (비활성 메뉴도 목록에 표시 가능)
- 기존 주문의 메뉴 참조(FK)는 유지됨

### BR-MENU-05: 메뉴 순서 관리
- sortOrder는 카테고리 내에서의 노출 순서
- 0 이상의 정수, 오름차순 정렬
- 새 메뉴 생성 시 sortOrder 미지정이면 해당 카테고리 내 최대값 + 1 자동 할당
- 순서 변경 API는 배열로 여러 메뉴의 순서를 한 번에 업데이트

### BR-MENU-06: 메뉴 소속 검증
- 메뉴 CRUD 시 storeId가 요청자의 매장과 일치하는지 검증
- 다른 매장의 메뉴 접근 시 404 반환

### BR-MENU-07: 카테고리 변경
- 메뉴 수정 시 categoryId 변경 가능
- 변경 대상 카테고리는 동일 매장 내 존재해야 함
- 카테고리 변경 시 새 카테고리 내 메뉴명 중복 검사 수행

## 3. 이미지 업로드 규칙

### BR-IMG-01: 파일 크기 제한
- 최대 5MB (5,242,880 bytes)
- 초과 시 에러: "이미지 파일 크기는 5MB 이하여야 합니다."

### BR-IMG-02: 파일 형식 제한
- 허용 MIME 타입: image/jpeg, image/png
- 허용 확장자: .jpg, .jpeg, .png
- 비허용 형식 시 에러: "JPG 또는 PNG 형식의 이미지만 업로드 가능합니다."
- MIME 타입과 확장자 모두 검증 (확장자 위조 방지)

### BR-IMG-03: 파일 저장 규칙
- 저장 경로: uploads/{storeId}/menus/
- 파일명: {menuId}_{timestamp}.{ext} (충돌 방지)
- 기존 이미지가 있으면 이전 파일 삭제 후 새 파일 저장
- imageUrl에는 상대 경로 저장 (예: /uploads/store1/menus/menu1_1710000000.jpg)

### BR-IMG-04: 이미지 없는 메뉴
- 이미지는 선택 사항 (imageUrl = null 허용)
- 이미지 없는 메뉴는 프론트엔드에서 기본 플레이스홀더 표시

## 4. 접근 제어 규칙

### BR-AUTH-01: 고객 접근 범위
- 메뉴 목록 조회 (GET /menus): 인증된 고객 (테이블 로그인)
- 메뉴 상세 조회 (GET /menus/:menuId): 인증된 고객
- 고객은 isActive=true인 메뉴만 조회 가능

### BR-AUTH-02: 관리자 접근 범위
- 카테고리 CRUD: 관리자 인증 필요
- 메뉴 CRUD: 관리자 인증 필요
- 이미지 업로드: 관리자 인증 필요
- 메뉴 순서 변경: 관리자 인증 필요
- 관리자는 isActive 상태와 무관하게 모든 메뉴 조회 가능

---

## 5. 데이터 무결성 규칙

### BR-INT-01: 멀티 테넌트 격리
- 모든 쿼리에 storeId 조건 필수
- 매장 간 데이터 접근 불가

### BR-INT-02: 참조 무결성
- Menu.categoryId는 반드시 존재하는 Category.id를 참조
- 카테고리 삭제 시 소속 활성 메뉴 존재 여부 확인 (BR-CAT-02)

### BR-INT-03: Soft Delete 일관성
- isActive=false인 메뉴는 고객 조회에서 제외
- 주문 시 메뉴 유효성 검증에서 isActive=true 확인 (Order Unit 책임)
